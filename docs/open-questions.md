# Open Questions

The following questions are intentionally unresolved in v0.1.0. They should become tracked issues before v0.5.0.

1. Which core artifact classes require canonical machine-readable schemas?
2. Should decision receipts be mandatory for every consequential effect or only profile-defined classes?
3. What minimum semantics are required to prove delegation scope attenuation across heterogeneous systems?
4. How should assurance levels be represented without creating false comparability across domains?
5. Which accountability obligations can be transferred, and which must remain non-delegable?
6. How should privacy-preserving graph traversal expose sufficient provenance without disclosing unrelated relationships?
7. When does a model, toolchain, memory or operator change create a new agent governance identity?
8. How should systemic and cumulative harms be connected to transaction-level effect records?
9. What evidence should be required before two independent implementations may claim profile interoperability?
10. Should a future release define a common policy-decision outcome vocabulary beyond permit, deny, restrict, suspend and review?
