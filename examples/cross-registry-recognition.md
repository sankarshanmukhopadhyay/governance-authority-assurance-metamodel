# Example: Cross-Registry Recognition

Registry A discovers an organisation record in Registry B. Technical interoperability allows the record to be retrieved, but Registry A recognises only selected record classes from B and only for low-risk supplier onboarding.

The recognition policy specifies accepted classes, purpose, assurance threshold, jurisdiction, freshness, revocation source and withdrawal conditions. Registry A’s relying service evaluates the record under its own policy. The discovered path is evidence, not an automatic trust decision.
