# v0.9.0 — Candidate Specification

GAAM v0.9.0 stabilises the normative and machine-readable surface for implementation, interoperability and assurance review. It corrects release identity drift, establishes canonical schema identifiers, strengthens profile and claim semantics, adds behavioural governance tests, introduces threat traceability, and produces a reproducible governance package.

## Material changes

- Candidate-stable normative artifact precedence and authority invariants.
- Eight aligned profile documents and manifests.
- Versioned canonical schema namespace and schema catalogue.
- Governed vocabularies with extension policies.
- L0–L4 evidence and independence rules.
- Behavioural vectors for authority, delegation, decisions and remedy.
- Threat-to-requirement-to-test traceability.
- Correct canonical TSMM repository reference with informative-boundary treatment.
- Real GitHub Actions validation workflow and hardened Pages navigation.
- Deterministic v0.9.0 package manifest and SHA-256 checksums.

## Limitations

Candidate status is not certification, universal assurance or independent interoperability. Independent implementation reports and candidate review findings remain prerequisites for v1.0.0.
