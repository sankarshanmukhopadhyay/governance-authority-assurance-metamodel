# Project Roadmap

The roadmap is directional. Issues and implementation evidence may change sequencing.

## v0.1.0 — Initial Public Draft

Establish the complete architecture, initial normative requirement set, conformance profiles and review package.

## v0.2.x–v0.4.x — Iterative refinement

Resolve terminology conflicts, strengthen lifecycle semantics, add implementation examples, test graph and delegation invariants, and refine machine-actionable artifact requirements.

## v0.5.0 — Feature Complete Draft

Include all planned concept classes, profiles, conformance targets and required companion guidance. New architectural scope should require exceptional justification after this milestone.

## v0.6.x–v0.8.x — Validation cycle

Collect implementation reports, threat-model findings, cross-ecosystem mappings and interoperability feedback. Stabilise normative text and schemas.

## v0.9.0 — Candidate Specification

Freeze the architecture and open a release-candidate review focused on defects, ambiguity, implementability and conformance.

## v1.0.0 — Stable Initial Release

Publish a reference-quality specification supported by implementation evidence, stable identifiers, conformance tests and resolved critical issues.
