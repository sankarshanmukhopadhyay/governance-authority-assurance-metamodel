# v0.5.0 — Feature Complete Draft

**Release date:** 2026-07-16

GAAM v0.5.0 completes the planned functional and architectural surface for implementation and structured validation. It does not claim specification stability, universal assurance, independent interoperability or production certification.

## Highlights

- Canonical authority, delegation, evidence, assurance, runtime, receipt, appeal, remedy, agent identity, profile, claim and package schemas.
- Explicit lifecycle transition and propagation semantics.
- Eight composable profiles with machine-readable manifests and dependency closure.
- Contextual assurance semantics that prohibit false cross-domain equivalence.
- Expanded privacy, safety, affected-party, systemic-harm and market-integrity requirements.
- Three complete implementation patterns with valid and invalid fixtures.
- Threat model, architectural decisions and generated validation evidence.
- GitHub Pages-oriented information architecture and AI/LLM usage disclosure.

## Compatibility posture

This is a draft release. Normative and schema changes remain possible during the validation cycle. Claims must bind to GAAM `0.5.0` and the exact profile versions used.

## Residual risks

- No independent implementation or interoperability report is included.
- Threat controls have not received external adversarial review.
- High-impact and systemic-harm thresholds remain context and profile defined.
- GitHub Pages deployment depends on repository settings enabling the provided workflow or Pages source.

## Validation

Run `python scripts/validate.py`. The generated [`VALIDATION_REPORT.md`](../VALIDATION_REPORT.md) and `validation/validation-report.json` provide release evidence.
